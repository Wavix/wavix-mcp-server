# Contributions

This repository is **source-available** under the [MIT License](LICENSE), but it is **not open to external contributions**. The Wavix MCP server is maintained by Wavix engineering, and the public catalogue you see here mirrors the canonical source maintained internally.

## What this means

- **Pull requests are not accepted.** Any PR opened against this repository will be closed automatically. You are welcome to fork the repository under the terms of the MIT License, but changes will not be merged upstream.
- **GitHub Issues and Discussions are disabled.** This keeps the repository as a read-only documentation surface.
- **There is no public roadmap or backlog here.** Roadmap decisions are driven by the Wavix product team.

## How to give feedback

We genuinely want to hear about bugs, missing examples, unclear instructions, or feature requests. The single supported channel is:

- **Email** — <support@wavix.com>

Useful things to include:

- Where you ran into the issue (file path / section of the documentation, or which MCP client).
- A short reproduction or quoted text.
- What you expected to see.

We review every message and roll fixes into the next documentation release.

## Security

Vulnerabilities go through a separate flow described in [`SECURITY.md`](SECURITY.md) — please follow that file rather than this one.

## Forks

You may fork the repository, modify it, and redistribute it under the MIT terms. Please:

- Remove or replace Wavix branding (logos, product names framed as endorsement) in any fork you publish.
- Do not present your fork as the official Wavix MCP server. The official endpoint remains `https://mcp.wavix.com/mcp`.
